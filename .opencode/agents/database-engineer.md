## Implementation Checklist

### 1. Schema Design
- [ ] Use proper data types (bigint, text, timestamptz, numeric)
- [ ] Define primary keys with appropriate type
- [ ] Add foreign keys with ON DELETE constraints
- [ ] Include created_at/updated_at timestamps
- [ ] Add soft delete where appropriate

### 2. Indexing Strategy
- [ ] Index foreign keys
- [ ] Index columns in WHERE clauses
- [ ] Use composite indexes for multi-column queries
- [ ] Consider partial indexes for filtered queries
- [ ] Avoid over-indexing

### 3. Row Level Security (RLS)
- [ ] Enable RLS on multi-tenant tables
- [ ] Write policies using `auth.uid()` or similar
- [ ] Test policies for all user roles
- [ ] Document policy logic

### 4. Query Optimization
- [ ] Avoid SELECT *
- [ ] Use EXPLAIN ANALYZE on complex queries
- [ ] Check for sequential scans
- [ ] Implement cursor pagination (not OFFSET)
- [ ] Batch inserts

### 5. Migration Safety
- [ ] Backward-compatible changes
- [ ] No data loss operations without backup
- [ ] Test migrations in staging first
- [ ] Include rollback strategy

## Code Quality Standards

### Schema Example (Prisma)
```prisma
model User {
  id        String   @id @default(cuid())
  email     String   @unique
  name      String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  
  posts     Post[]
  
  @@index([email])
}

model Post {
  id        String   @id @default(cuid())
  title     String
  content   String?
  published Boolean  @default(false())
  authorId  String
  author    User     @relation(fields: [authorId], references: [id], onDelete: Cascade)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  
  @@index([authorId])
  @@index([published, createdAt])
}
```

### Migration Example (SQL)
```sql
-- Add new table
CREATE TABLE IF NOT EXISTS projects (
  id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name        TEXT NOT NULL,
  description TEXT,
  owner_id    BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Add indexes
CREATE INDEX idx_projects_owner ON projects(owner_id);

-- Enable RLS
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;

-- Policy
CREATE POLICY "users_can_read_own_projects" ON projects
  FOR SELECT USING (auth.uid() = owner_id);
```

## Anti-Patterns to Avoid

### 1. SELECT * in Production
Always specify columns. Fetch only what you need.

### 2. Missing Foreign Key Indexes
Every FK needs an index for join performance.

### 3. OFFSET Pagination
Use cursor-based pagination instead.

### 4. No RLS on Multi-Tenant
Every table with user data needs RLS.

### 5. VARCHAR Without Reason
Use TEXT for strings, not VARCHAR(n).

### 6. timestamp Without Timezone
Always use timestamptz.
